## Findings considered and rejected

- **`huf/ai/flow_eval.py` `safe_eval_expression` is secure by design — NOT a finding.** The AST evaluator rejects function calls (`ast.Call`), attribute access (`ast.Attribute`), and any unsupported node, allows only a whitelisted operator set, binds only `context`, and caps length at 500 chars. No code-injection path was found by reading. Plan 001 adds regression tests for it (defense in depth), but there is no vulnerability to fix here. Do not re-audit as a security gap.
- **The ~22 `except Exception: pass` blocks are mostly graceful degradation — NOT planned.** Examples include `flow_api.py:374` (payload parse fallback) and similar best-effort paths. They suppress non-security errors where a sensible default exists. Broadly tightening them is a code-quality task with low security value and high churn; left unplanned. (If a future audit finds one that swallows an auth/permission error, that specific one should be reopened.)
- **`MAX_RESPONSE_SIZE` memory pre-check gap in `http_handler.py` — known minor gap, left unplanned.** The handler checks `Content-Length` then `len(response.content)` (http_handler.py:131-146), but `response.content` materializes the full body in memory BEFORE the size check, so a server that omits/lies about `Content-Length` and streams a huge body could exhaust memory. The proper fix is streaming with `stream=True` and incremental size accumulation. Judged minor (10MB cap, internal tool, requires a hostile but reachable public host) and deliberately deferred — noted here so it is not re-discovered as novel.
- **DNS rebinding / TOCTOU in `validate_url` — partially deferred, documented in plan 002.** `validate_url` resolves DNS once and `requests` re-resolves at fetch time, so the line-33 "prevents DNS rebinding" claim is false. Plan 002 corrects the comment and closes the IPv6/redirect holes but does NOT pin the resolved IP into the request (the full rebinding fix). This is a deliberate, documented residual in 002's Maintenance notes, not an oversight.
- **Hardcoded-URL integration tools (`huf/ai/tools/google_*`, slack, gmail, etc.) — out of scope, no SSRF surface.** They call fixed vendor API hosts, not user-supplied URLs, so the SSRF guard does not apply. Explicitly excluded from plan 002.
