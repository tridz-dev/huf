# Installation

Huf is a Frappe application that can be installed on any Frappe site. This guide will walk you through the installation process, whether you're using Frappe Bench (for local development or self-hosted setups) or Frappe Cloud.

## Prerequisites

Before installing Huf, you need:

- A Frappe site (either local via Bench or on Frappe Cloud)
- Python 3.8 or higher
- Access to your site's terminal/command line

## What is Frappe Bench?

**Frappe Bench** is a command-line tool that helps you manage Frappe applications and sites. If you're new to Frappe, think of Bench as a package manager for Frappe apps - it helps you install, update, and manage multiple Frappe applications on your system.

If you don't have Bench installed yet, you can learn more at [frappeframework.com](https://frappeframework.com/docs/user/en/installation).

## Installation on Frappe Bench

If you're running Frappe locally or on your own server using Bench, follow these steps:

### Step 1: Navigate to Your Bench Directory

Open your terminal and navigate to your Bench directory. This is typically where you initialized Bench:

```bash
cd $PATH_TO_YOUR_BENCH
```

For example, if your Bench is at `/home/user/frappe-bench`, you would run:

```bash
cd /home/user/frappe-bench
```

### Step 2: Get the Huf App

Use Bench to download the Huf application from GitHub:

```bash
bench get-app https://github.com/Tridz/agentflo.git
```

This command downloads Huf into your Bench's `apps` directory.

### Step 3: Install Huf on Your Site

Install Huf on your Frappe site. Replace `your-site-name` with your actual site name:

```bash
bench --site your-site-name install-app agentflo
```

### Step 4: Install Dependencies

Huf requires some Python packages (including LiteLLM for AI provider support). Install them:

```bash
bench setup requirements
```

This command installs all required dependencies for all apps in your Bench, including Huf.

### Step 5: Restart Your Site

After installation, restart your site to ensure all changes are loaded:

```bash
bench restart
```

### Step 6: Access Huf

1. Log in to your Frappe site
2. Navigate to the App Switcher (grid icon in the top right)
3. Look for "Huf" or "AgentFlo" in the app list
4. Click on it to access the Huf interface

## Installation on Frappe Cloud

Frappe Cloud is a managed hosting platform for Frappe applications. If your site is hosted on Frappe Cloud, the installation process is simpler:

### Step 1: Access Your Site's Console

1. Log in to [frappecloud.com](https://frappecloud.com)
2. Navigate to your site
3. Go to **Settings** → **Console** (or use the terminal icon)

### Step 2: Install the App

In the console, run:

```bash
bench get-app https://github.com/Tridz/agentflo.git
bench --site $CI_COMMIT_REF_NAME install-app agentflo
bench setup requirements
bench restart
```

> **Note**: On Frappe Cloud, `$CI_COMMIT_REF_NAME` is automatically set to your site name, so you don't need to replace it.

### Step 3: Access Huf

After installation completes:
1. Refresh your site
2. Go to the App Switcher
3. Click on "Huf" or "AgentFlo"

## Verifying Installation

To verify that Huf is installed correctly:

1. Check that you can see "Huf" or "AgentFlo" in your App Switcher
2. Try creating a new **AI Provider** document (you should see it in the Huf app)
3. Check the console for any error messages

## Troubleshooting

### "Command not found: bench"

If you see this error, Bench is not installed or not in your PATH. See the [Frappe installation guide](https://frappeframework.com/docs/user/en/installation) for help setting up Bench.

### "App not found" or Installation Errors

- Make sure you're in the correct Bench directory
- Verify your site name is correct
- Check that you have internet connectivity (needed to download from GitHub)
- Try running `bench update` first to ensure your Bench is up to date

### Dependencies Not Installing

If `bench setup requirements` fails:
- Check your Python version (should be 3.8+)
- Ensure you have pip installed
- Try running `bench setup requirements --dev` if you're in a development environment

### LiteLLM Not Found

Huf uses LiteLLM for AI provider support. If you see errors about LiteLLM:
- Run `bench setup requirements` again
- Check that the installation completed successfully
- Restart your site with `bench restart`
