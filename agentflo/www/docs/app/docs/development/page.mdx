# Development Guide

This section is for developers who want to understand Huf's architecture, extend its functionality, or contribute to the project.

## Project Structure

Huf follows the standard Frappe application structure:

```
agentflo/
├── agentflo/                    # Main Python module
│   ├── __init__.py
│   └── doctype/                 # DocType definitions
│       ├── agent/
│       ├── ai_provider/
│       ├── ai_model/
│       ├── agent_tool_function/
│       └── ...                  # Other DocTypes
├── ai/                          # Core AI integration logic
│   ├── agent_integration.py    # Main agent management
│   ├── conversation_manager.py # Conversation handling
│   ├── sdk_tools.py            # Tool serialization
│   ├── tool_functions.py       # Low-level tool functions
│   ├── run.py                  # Provider routing
│   └── providers/              # Provider implementations
│       └── litellm.py          # LiteLLM provider
├── hooks.py                    # Frappe hooks
├── install.py                  # Installation hooks
└── www/                        # Web assets
    └── docs/                   # This documentation
```

## Core Components

### DocTypes

Huf uses several DocTypes to manage agents and their configuration:

- **AI Provider**: Stores API credentials for AI services
- **AI Model**: Defines models available from providers
- **Agent Tool Function**: Defines tools agents can use
- **Agent**: The main agent configuration
- **Agent Conversation**: Tracks conversation sessions
- **Agent Message**: Individual messages in conversations
- **Agent Run**: Logs agent executions
- **Agent Tool Call**: Records tool usage

Each DocType follows Frappe conventions:
- `<doctype_name>/<doctype_name>.json`: Schema definition
- `<doctype_name>/<doctype_name>.py`: Server-side controller
- `<doctype_name>/<doctype_name>.js`: Client-side script (if needed)

### AI Integration (`ai/` directory)

The `ai/` directory contains the core logic:

- **`agent_integration.py`**: `AgentManager` class that prepares agents for execution
- **`conversation_manager.py`**: Handles conversation persistence and history
- **`sdk_tools.py`**: Bridges Huf tools to the agents SDK format
- **`tool_functions.py`**: Low-level functions for Frappe operations
- **`run.py`**: Routes provider requests to appropriate implementations
- **`providers/litellm.py`**: Unified provider using LiteLLM

## Frappe Framework Integration

Huf is built on the [Frappe Framework](https://frappeframework.com/docs), which means:

### Standard Frappe Features

- **DocTypes**: All data models use Frappe DocTypes
- **Permissions**: Standard Frappe permission system
- **Hooks**: Uses Frappe hooks for extensibility
- **API**: Standard Frappe API endpoints
- **Client Scripts**: Standard client-side scripting

### Frappe Enhancements

Since Huf is a Frappe app, all Frappe enhancement mechanisms work:

- **Custom Fields**: Add custom fields to any DocType
- **Custom Scripts**: Add client/server scripts
- **Hooks**: Extend functionality via hooks
- **Overrides**: Override methods and templates
- **Custom Apps**: Create apps that extend Huf

For more information, see the [Frappe Documentation](https://frappeframework.com/docs).

## Extending Huf

### Custom Tool Functions

Create custom Python functions that agents can call:

1. **Create the Function**

```python
# my_app/api.py
import frappe

@frappe.whitelist()
def calculate_discount(item_code, quantity):
    """Calculate discount for an item based on quantity"""
    # Your custom logic here
    if quantity > 100:
        return 0.15  # 15% discount
    elif quantity > 50:
        return 0.10  # 10% discount
    return 0.05  # 5% discount
```

2. **Create Tool Function DocType**

- Go to **Agent Tool Function**
- Create new tool
- Set **Types** to "Custom Function"
- Set **Function Path** to `my_app.api.calculate_discount`
- Define parameters

3. **Add to Agent**

- Add the tool function to your agent's tool list

### Custom Providers

While Huf uses LiteLLM for most providers, you can add custom providers:

1. **Create Provider Module**

```python
# agentflo/ai/providers/custom.py
async def run(agent, enhanced_prompt, provider, model, context=None):
    # Your custom provider implementation
    pass
```

2. **Register in `run.py`**

Add routing logic to direct requests to your provider.

### Hooks

Use Frappe hooks to extend Huf:

**`hooks.py`:**
```python
# Add custom validations
agent_before_save = "my_app.hooks.validate_agent"

# Add custom tool handlers
agent_tool_call = "my_app.hooks.handle_tool_call"
```

## Development Setup

1. **Clone the Repository**

```bash
git clone https://github.com/Tridz/agentflo.git
```

2. **Install in Bench**

```bash
cd /path/to/bench
bench get-app agentflo /path/to/agentflo
bench --site your-site install-app agentflo
bench setup requirements
```

3. **Development Mode**

```bash
bench start
```

4. **Run Tests** (if available)

```bash
bench --site your-site run-tests --app agentflo
```

## Architecture Decisions

### Why LiteLLM?

LiteLLM provides:
- Unified interface to 100+ LLM providers
- Automatic retry logic
- Cost tracking
- Model name normalization
- Built-in error handling

### Why Frappe DocTypes?

DocTypes provide:
- Built-in CRUD operations
- Permission system
- Audit trails
- Standard UI
- Database abstraction

### Tool System Design

Tools are designed to be:
- **Flexible**: Support multiple tool types
- **Extensible**: Easy to add custom tools
- **Secure**: Respect Frappe permissions
- **Trackable**: All tool calls are logged
