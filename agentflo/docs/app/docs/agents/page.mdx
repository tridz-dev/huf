# Agents

Agents are the core of Huf. An agent is an AI-powered assistant that can understand instructions, use tools to interact with your Frappe site, and respond intelligently to user requests.

## Understanding Agents

An agent in Huf consists of:

- **Instructions**: The system prompt that defines the agent's personality, goals, and behavior
- **AI Model**: The underlying AI model that powers the agent (e.g., GPT-4, Claude, Gemini)
- **Tools**: The capabilities the agent can use to interact with your site
- **Settings**: Configuration like temperature and top_p that control the AI's behavior

## Creating an Agent

### Basic Configuration

1. **Navigate to Agent**
   - Open the Huf app and click **Agent** in the sidebar

2. **Create New Agent**
   - Click **New** to create a new agent

3. **Required Fields**
   - **Agent Name**: A unique, descriptive name for your agent
   - **Provider**: Select the AI Provider you've configured
   - **Model**: Select the AI Model you want to use

4. **Instructions**
   - Write clear, detailed instructions for your agent
   - Be specific about the agent's role, tone, and constraints
   - Example:
     ```
     You are a customer service assistant for an e-commerce platform. 
     Your role is to help customers with order inquiries, product questions, 
     and account issues. Always be polite, professional, and helpful. 
     When you don't know something, admit it and offer to find the information.
     ```

5. **Model Settings**
   - **Temperature** (0.0 - 2.0): Controls randomness
     - Lower (0.0-0.3): More focused and deterministic
     - Medium (0.7): Balanced creativity and consistency
     - Higher (1.0-2.0): More creative and varied responses
   - **Top P** (0.0 - 1.0): Alternative to temperature
     - Controls diversity via nucleus sampling
     - Start with 1.0 for most use cases

6. **Save Your Agent**

## Agent Types

Huf supports three main ways to use agents:

### 1. Chat Agents

Chat agents provide an interactive conversation interface.

**To Enable Chat:**
- Check the **Enable Chat** checkbox when creating/editing an agent
- Users can then interact with the agent through the **Agent Chat** interface
- Conversations are automatically tracked and maintained

**Use Cases:**
- Customer support chatbots
- Internal knowledge assistants
- Interactive data exploration tools

### 2. Event-Driven Agents

Event-driven agents automatically run when specific events occur in your Frappe site.

**How to Configure:**
1. Create or edit an agent
2. Go to **Agent Trigger** section
3. Add a trigger with:
   - **DocType**: The document type to monitor (e.g., "Sales Order")
   - **Event**: When to trigger (e.g., "on_submit", "after_insert")
   - **Condition** (optional): Python expression to filter when the agent runs
     - Example: `doc.grand_total > 10000` (only run for orders over $10,000)

**Use Cases:**
- Automatically validate documents before submission
- Send notifications when certain conditions are met
- Auto-populate fields based on AI analysis

**Example Condition:**
```python
doc.status == "Pending" and doc.grand_total > 5000
```

This will only trigger the agent for pending documents with a grand total over 5000.

### 3. Scheduled Agents

Scheduled agents run automatically at specified intervals.

**How to Configure:**
1. Create or edit an agent
2. Go to **Agent Trigger** section
3. Add a scheduled trigger with:
   - **Frequency**: How often to run (hourly, daily, weekly, etc.)
   - **Time**: When to run (for daily/weekly schedules)

**Use Cases:**
- Daily reports and summaries
- Periodic data analysis
- Automated maintenance tasks
- Regular data synchronization

## Adding Tools to Agents

Tools give agents the ability to interact with your Frappe site. To add tools:

1. **Create Tool Functions** (if needed)
   - Go to **Agent Tool Function** and create the tools you need
   - See the [Tools Guide](/docs/tools) for details

2. **Add Tools to Agent**
   - Edit your agent
   - Scroll to the **Agent Tool** table
   - Click **Add Row**
   - Select a tool function from the dropdown
   - Save the agent

**Best Practices:**
- Start with a few essential tools and add more as needed
- Give tools clear, descriptive names
- Write detailed tool descriptions - the AI uses these to decide when to use tools

## Agent Instructions Best Practices

Good instructions are crucial for agent performance:

### Be Specific
❌ "Help with customer service"  
✅ "You are a customer service assistant. Help customers with order tracking, returns, and product questions. Always verify order numbers before providing information."

### Set Boundaries
❌ "Answer questions"  
✅ "Answer questions about orders, products, and accounts. Do not provide refunds or make changes to orders - direct customers to support for those actions."

### Define Tone and Style
❌ "Be professional"  
✅ "Use a friendly, professional tone. Keep responses concise (2-3 sentences when possible). Use the customer's name when available."

### Provide Examples
```
When a customer asks about order status:
1. Ask for their order number
2. Use the get_order tool to retrieve order details
3. Provide a clear status update with expected delivery date
```

## Managing Agents

### Viewing All Agents
- Go to **Agent** list view to see all your agents
- Use filters and search to find specific agents

### Editing Agents
- Click on an agent to edit it
- Changes take effect immediately for new conversations/runs

### Testing Agents
- Use **Agent Console** to test agents quickly
- Check **Agent Run** to see execution logs and debug issues

### Monitoring Performance
- View **Agent Run** documents to see:
  - Success/failure rates
  - Token usage and costs
  - Response times
  - Error messages

## Common Agent Patterns

### Customer Support Agent
- **Tools**: Get Customer, Get Order, Create Ticket
- **Instructions**: Focus on helping customers with common issues
- **Type**: Chat agent

### Data Entry Assistant
- **Tools**: Create Document, Update Document, Get List
- **Instructions**: Help users create and update records accurately
- **Type**: Chat agent or event-driven

### Report Generator
- **Tools**: Get List (with filters), Custom analysis functions
- **Instructions**: Generate summaries and insights from data
- **Type**: Scheduled agent

### Document Validator
- **Tools**: Get Document, Update Document
- **Instructions**: Validate document data and flag issues
- **Type**: Event-driven agent (on_submit)

## Troubleshooting

### Agent Not Responding
- Check API key validity and credits
- Verify model name is correct
- Review **Agent Run** for error messages
- Ensure provider is properly configured

### Agent Using Wrong Tools
- Improve tool descriptions
- Refine agent instructions to be more specific about tool usage
- Review tool call logs in **Agent Run**

### High Token Usage
- Use smaller models for simple tasks
- Reduce conversation history length
- Optimize instructions to be more concise
- Consider using cheaper models for testing

### Agent Not Following Instructions
- Make instructions more explicit
- Add examples of desired behavior
- Adjust temperature (lower = more consistent)
- Review and refine instructions based on actual behavior
