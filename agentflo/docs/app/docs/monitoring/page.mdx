# Monitoring and Analytics

Huf provides comprehensive monitoring and analytics to help you track agent performance, understand usage patterns, and manage costs.

## Agent Run

**Agent Run** is the primary way to monitor agent executions. Every time an agent runs (whether triggered manually, by schedule, or by event), an Agent Run document is created.

### Viewing Agent Runs

1. Navigate to **Agent Run** in the Huf sidebar
2. View the list of all agent executions
3. Use filters to find specific runs:
   - By agent
   - By status (Success, Failed, Queued)
   - By date range
   - By conversation

### Agent Run Details

Click on an Agent Run to see:

- **Status**: Whether the run succeeded or failed
- **Agent**: Which agent was executed
- **Prompt**: The user's input that triggered the run
- **Response**: The agent's final response
- **Error Message**: Any errors that occurred (if failed)
- **Token Usage**:
  - Input tokens (prompt + context)
  - Output tokens (response)
  - Total tokens
- **Cost**: Estimated cost based on token usage and model pricing
- **Tool Calls**: Which tools were used and their results
- **Timing**: When the run started and completed

### Understanding Status

- **Started**: Run has begun processing
- **Queued**: Run is waiting to be processed
- **Success**: Run completed successfully
- **Failed**: Run encountered an error

### Analyzing Agent Runs

Use Agent Run data to:
- **Identify Issues**: Find failed runs and understand why they failed
- **Track Performance**: Monitor response times and success rates
- **Optimize Costs**: Review token usage and costs per run
- **Improve Agents**: See which prompts work best and refine instructions

## Token Usage and Costs

### Understanding Tokens

Tokens are the units that AI models use to process text:
- **Input Tokens**: Text sent to the model (prompt + conversation history)
- **Output Tokens**: Text generated by the model (response)
- **Total Tokens**: Sum of input and output tokens

### Cost Calculation

Huf automatically calculates costs based on:
- Token usage (input and output)
- Model pricing (varies by provider and model)
- Current market rates

**Note**: Cost estimates are approximate and may vary based on:
- Provider pricing changes
- Model-specific pricing
- Regional variations

### Monitoring Costs

1. **Per-Run Costs**: Check individual Agent Run documents
2. **Aggregate Costs**: Use list view filters and grouping
3. **Cost Trends**: Track costs over time to identify patterns
4. **Model Comparison**: Compare costs across different models

### Cost Optimization Tips

- **Use Appropriate Models**: Smaller models for simple tasks, larger for complex ones
- **Optimize Prompts**: Shorter, more focused prompts use fewer tokens
- **Limit Context**: Reduce conversation history length when possible
- **Monitor Regularly**: Keep an eye on costs and adjust as needed

## Agent Tool Call

**Agent Tool Call** tracks every time an agent uses a tool. This provides detailed visibility into:
- Which tools are being used
- How often tools are called
- What parameters are passed to tools
- What results tools return
- Any errors that occur

### Viewing Tool Calls

1. Navigate to **Agent Tool Call** in the sidebar
2. View all tool calls across all agents
3. Filter by:
   - Agent Run
   - Tool function
   - Status (success/failure)
   - Date range

### Tool Call Details

Each tool call shows:
- **Tool**: Which tool was called
- **Arguments**: Parameters passed to the tool
- **Result**: What the tool returned
- **Error**: Any errors that occurred
- **Timing**: When the tool was called

### Analyzing Tool Usage

Use tool call data to:
- **Identify Popular Tools**: See which tools are used most
- **Optimize Tools**: Improve tools that are frequently used
- **Debug Issues**: Find tools that frequently fail
- **Understand Agent Behavior**: See how agents use tools to solve problems

## Conversation Analytics

### Agent Conversation

**Agent Conversation** provides high-level conversation metrics:
- **Total Messages**: Number of messages in the conversation
- **Is Active**: Whether the conversation is ongoing
- **Agent**: Which agent is handling the conversation
- **Title**: Auto-generated or custom conversation title

### Agent Message

**Agent Message** shows individual messages:
- **Role**: User, agent, or system message
- **Content**: The message text
- **Kind**: Message, tool call, tool result, or error
- **Run**: Associated Agent Run (for agent messages)

### Analyzing Conversations

Use conversation data to:
- **Track Engagement**: See how many messages per conversation
- **Identify Patterns**: Find common questions or topics
- **Improve Agents**: Understand what users are asking for
- **Monitor Quality**: Review agent responses for accuracy

## Best Practices

### Regular Monitoring

- **Daily**: Check for failed runs and errors
- **Weekly**: Review costs and usage trends
- **Monthly**: Analyze performance and optimize

### Setting Up Alerts

Consider setting up alerts for:
- High failure rates
- Unusual cost spikes
- Tool errors
- Performance degradation

### Documentation

- Document common issues and solutions
- Keep notes on agent improvements
- Track changes and their impact
- Share insights with your team

## Troubleshooting

### High Costs

- Review token usage in Agent Run
- Check for unnecessarily long prompts
- Consider using cheaper models for simple tasks
- Optimize conversation history length

### Frequent Failures

- Check error messages in Agent Run
- Review tool call errors
- Verify API keys and provider status
- Test agents in Agent Console

### Performance Issues

- Monitor response times
- Check for slow tools
- Optimize tool functions
- Consider model selection

### Missing Data

- Verify agents are running
- Check Agent Run creation
- Review conversation tracking
- Ensure proper permissions
