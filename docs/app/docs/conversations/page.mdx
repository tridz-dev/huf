# Conversations

Conversations in Huf track the complete history of interactions between users and agents. This allows agents to maintain context across multiple messages and provide more natural, helpful responses.

## Understanding Conversations

A conversation in Huf:
- **Tracks Messages**: Stores every message exchanged between user and agent
- **Maintains Context**: Allows agents to remember previous messages in the session
- **Links to Agent Runs**: Each agent response is linked to an Agent Run for detailed tracking
- **Supports Multiple Sessions**: Users can have multiple active conversations with different agents

## How Conversations Work

### Automatic Creation

Conversations are automatically created when:
- A user starts chatting with a chat-enabled agent
- An agent is triggered via API with a session ID
- A scheduled or event-driven agent needs to maintain context

### Message Flow

1. **User Message**: User sends a message (via chat, API, or trigger)
2. **Conversation Lookup**: System finds or creates a conversation for the session
3. **Context Loading**: Agent loads recent messages from the conversation
4. **Agent Processing**: Agent processes the message with context
5. **Response**: Agent generates a response
6. **Message Storage**: Both user message and agent response are stored
7. **Context Update**: Conversation is updated with new messages

## Viewing Conversations

### Agent Chat Interface

For chat-enabled agents:
1. Navigate to **Agent Chat**
2. Select an agent
3. View the conversation history in the chat interface
4. Continue the conversation by typing new messages

### Agent Conversation List

To see all conversations:
1. Go to **Agent Conversation** in the sidebar
2. View the list of all conversations
3. Click on a conversation to see details:
   - Title
   - Associated agent
   - Total messages
   - Active status

### Individual Messages

To see individual messages:
1. Open an **Agent Conversation**
2. View the linked **Agent Message** documents
3. Each message shows:
   - Role (user, agent, or system)
   - Content
   - Timestamp
   - Associated Agent Run (for agent messages)

## Conversation Context

### How Much History?

By default, agents use recent conversation history to maintain context. The exact amount depends on:
- Model token limits
- Conversation length
- System configuration

Typically, agents use the last 10-20 messages to maintain context while staying within token limits.

### Context Window

The context window is the amount of conversation history the agent can "see":
- **Recent Messages**: Most recent messages are prioritized
- **Token Limits**: Models have maximum token limits
- **Automatic Truncation**: Older messages may be truncated if needed

## Managing Conversations

### Starting a New Conversation

**In Agent Chat:**
- Simply start typing - a new conversation is created automatically
- Or use the "New Conversation" button if available

**Via API:**
- Use a new session ID to start a fresh conversation
- Or explicitly create a new Agent Conversation document

### Continuing a Conversation

**In Agent Chat:**
- Your conversation continues automatically
- Previous messages are visible in the chat history

**Via API:**
- Use the same session ID to continue the conversation
- The system automatically loads previous messages

### Ending a Conversation

Conversations remain active until:
- Explicitly marked as inactive
- A new conversation is started with a different session ID
- The conversation is manually closed

## Conversation Best Practices

### For Users

1. **Be Clear**: Ask specific questions for better responses
2. **Provide Context**: Reference previous messages when needed
3. **Stay On Topic**: Keep conversations focused for better context retention
4. **Start Fresh**: Begin a new conversation for unrelated topics

### For Administrators

1. **Monitor Length**: Very long conversations may hit token limits
2. **Review Quality**: Check conversations to improve agent instructions
3. **Manage Storage**: Archive or delete old conversations if needed
4. **Privacy**: Be aware of what data is stored in conversations

## Conversation Types

### Chat Conversations

- Interactive, real-time conversations
- Best for customer support, Q&A, and assistance
- Maintains context throughout the session
- Accessible via Agent Chat interface

### API Conversations

- Programmatic conversations via API calls
- Useful for integrations and automation
- Controlled via session IDs
- Can be triggered from external systems

### Event-Driven Conversations

- Conversations triggered by document events
- May include document context in messages
- Useful for automated workflows
- Context maintained across multiple events

## Privacy and Security

### Data Storage

- All conversations are stored in your Frappe database
- Messages are linked to user accounts
- Follow your organization's data retention policies

### Access Control

- Conversations respect Frappe's permission system
- Users can only see conversations they're part of
- Administrators can view all conversations

### Sensitive Information

- Be cautious about storing sensitive data in conversations
- Consider what information agents have access to
- Review conversation logs regularly

## Troubleshooting

### Conversation Not Maintaining Context

- Check that messages are being saved correctly
- Verify the conversation is active
- Ensure the agent is loading conversation history
- Check Agent Run logs for errors

### Missing Messages

- Verify messages are being created in Agent Message
- Check that the conversation is linked correctly
- Review Agent Run status for failed runs

### Performance Issues

- Long conversations may slow down responses
- Consider archiving old conversations
- Monitor token usage in Agent Run

### Context Confusion

- Agents may get confused in very long conversations
- Start a new conversation for unrelated topics
- Review and refine agent instructions
